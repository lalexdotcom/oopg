    The current implementation:
    1. Uses `new Promise(async (res, rej) => ...)` on line 133 (BUG-01 anti-pattern)
    2. Calls `curs.close()` in both try and catch branches but not in a finally (BUG-02 cursor leak)
    3. Has a `promiseClose` that calls `res()` to settle the outer promise (artifact of the anti-pattern)

    Replace with this implementation body (after the argument parsing switch block and `if (!callback)` guard):

    ```typescript
    export async function chunk<R extends Row = Row>(
      client: ClientBase,
      sql: string,
      valuesOrCallback: unknown[] | ChunkCallback<R>,
      callbackOrOptions?: ChunkCallback<R> | ChunksOptions,
      optionsWhenValues?: ChunksOptions,
    ) {
      // ... existing argument parsing switch block (lines 114-127) stays exactly as-is ...
      if (!callback) throw new Error('No callback');
      const { size = 100, wait = true, ...cursorOptions } = options ?? {};

      const curs = cursor<R>(client, sql, values, cursorOptions);
      let position = 0;
      try {
        let rows: R[];
        // biome-ignore lint/suspicious/noAssignInExpressions: idiomatic cursor read pattern
        while ((rows = await curs.read(size)).length) {
          const nextPosition = position + rows.length;
          const close = async () => { await curs.close(); };
          if (wait) {
            await callback(rows, position, close);
          } else {
            callback(rows, position, close);
          }
          position = nextPosition;
        }
      } finally {
        await curs.close();
      }
    }
    ```

    Key changes:
    - Add `async` keyword to the implementation signature (third overload)
    - Remove the entire `return new Promise<void>(async (res, rej) => { ... })` wrapper
    - Remove the `// biome-ignore lint/suspicious/noAsyncPromiseExecutor` suppression comment (line 132)
    - Replace `promiseClose` with a simple `close` arrow function that only calls `await curs.close()` (no `res()` call)
    - Replace the `try { ... } catch (e) { await curs.close(); rej(e); }` with `try { ... } finally { await curs.close(); }`
    - Keep the `// biome-ignore lint/suspicious/noAssignInExpressions: idiomatic cursor read pattern` suppression on the while-loop (this is intentionally deferred per CONTEXT.md)
    - Remove `callback?.` optional chaining — callback is guaranteed non-null by the guard above, use `callback(...)` directly

    Do NOT modify `step()` or `first()` — they delegate to `chunk()` and get the fix transitively.
